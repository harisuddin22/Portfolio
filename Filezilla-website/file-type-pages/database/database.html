Database Formats

Relational database. This is a database that stores data in tables, which consist of rows and columns. Each row represents a record, and each column represents a field or attribute. Relational databases use a standard query language (SQL) to manipulate and retrieve data.

Document database. This is a database that stores data in documents, which are self-contained units of data that can have various structures and formats, such as JSON, XML, or BSON. Document databases are flexible and scalable, and can handle complex and nested data.

Key-value database. This is a database that stores data as pairs of keys and values, where each key is a unique identifier and each value is the data associated with that key. Key-value databases are simple and fast, and can handle large amounts of data.

Graph database. This is a database that stores data as nodes and edges, where each node represents an entity and each edge represents a relationship between entities. Graph databases are useful for modeling complex and interconnected data, such as social networks, recommendation systems, or fraud detection.

Object-oriented database. This is a database that stores data as objects, which are instances of classes that have attributes and methods. Object-oriented databases are compatible with object-oriented programming languages, such as Java, C++, or Python.

Hierarchical database. This is a database that stores data in a tree-like structure, where each node has a parent node and zero or more child nodes. Hierarchical databases are efficient and simple, and can handle hierarchical data, such as file systems, organizational charts, or XML documents.
